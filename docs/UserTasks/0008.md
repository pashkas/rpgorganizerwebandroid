# 0008 — Замена masonry на CSS Grid

## Запрос пользователя
«Есть идеи как улучшить масонри в главном окне?» → обсуждение → решили заменить на CSS Grid для сохранения порядка по строкам.

## Проблема
В главном окне использовались два разных подхода для masonry-раскладки:
- **Навыки** — библиотека `ngx-masonry` с JS-зависимостью (masonry-layout)
- **Квесты** — CSS `column-count`

Оба подхода раскладывали элементы **по столбцам** (сверху вниз), а не по строкам (слева направо). Пользователю нужен порядок по строкам — чтобы сортировка визуально соответствовала порядку данных.

Дополнительно: в `/pers` (квесты, инвентарь, достижения, лавка) и в диалоге выбора картинки тоже использовался `column-count`.

## Решение
Заменили всё на **CSS Grid** (`display: grid; grid-template-columns: repeat(N, 1fr)`):
- Порядок по строкам — из коробки
- Адаптивность через media queries (3 колонки мобилка, 5 средний, 8 большой)
- Нет JS-зависимости
- Единый класс `.masonry-grid` для всех мест

## Что сделано
1. **HTML main-window** — заменили `<ngx-masonry>` (навыки) и `column-count` div (квесты) на `<div class="masonry-grid">`
2. **HTML pers-list** — заменили `masonry-list` на `masonry-grid` в 4 местах (квесты, инвентарь, достижения, лавка)
3. **HTML add-item-dialog** — заменили `masonry-list-galery` на `masonry-grid` в галерее картинок
4. **CSS styles.css** — удалены классы: `.ngx-masonry-item*`, `.main-masonry*`, `.masonry-list`, `.masonry-list-galery`, `.masonry-item`. Добавлен `.masonry-grid` с 3 media queries
5. **TS main-window.component.ts** — убраны: `NgxMasonryComponent`, `ViewChild`, `BreakpointObserver`, `itemsLoaded()`, `myOptions`, `globalMasonryCls$`, `globalMasonryClsNGX$`
6. **SharedModule** — убраны `NgxMasonryModule` и `MasonryPipe` из declarations/imports/exports
7. **Удалены файлы:** `masonry.pipe.ts`, `masonry.pipe.spec.ts` (мёртвый код — pipe с недостижимым кодом после return)

## Побочные эффекты
- Библиотека `ngx-masonry` больше не используется в коде, но остаётся в `package.json` — можно удалить при следующем `npm install`
- Все ячейки в строке теперь одной высоты (CSS Grid vs true masonry). Для текущих данных (квадратные иконки навыков, похожие по размеру карточки квестов/наград) это не проблема
