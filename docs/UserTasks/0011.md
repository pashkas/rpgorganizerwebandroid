# 0011 — Унификация отображения картинок в карточках masonry-grid

## Запрос пользователя
«Посмотри масонри в главном окне. Там картинка посередине если не квадратная, а текст внизу. А в /pers по другому. Там картинка вверху. Может одинаково сделаем?» — хочет как в main-window (картинка по центру).

## Проблема
В main-window карточки (квесты, навыки) использовали класс `masonry-img` (height: 100%, width: 100%, object-fit: contain) — картинка заполняла ячейку и центрировалась. В `/pers` (квесты, инвентарь, достижения, лавка) использовался `img-fluid rounded w-100` с inline `object-fit: contain` — картинка сохраняла натуральные пропорции и прижималась к верху.

## Решение
Заменили `class="img-fluid rounded w-100" [ngStyle]="{ 'object-fit': 'contain' }"` на `class="masonry-img"` во всех карточках `/pers`:

1. **Квесты** (вкладка квестов) — `pers-list.component.html`, строка ~348
2. **Инвентарь** (вкладка наград → Инвентарь) — строка ~428
3. **Достижения** (вкладка наград → Достижения) — строка ~462
4. **Лавка** (вкладка наград → Лавка) — строка ~498

## Затронутые файлы
- `src/app/pers-list/pers-list.component.html` — 4 замены класса картинок

## Корректировки от пользователя
- Изначально я сделал наоборот (main-window под pers) — пользователь уточнил, что хочет как в main-window (картинка по центру, текст внизу). Откатил и сделал правильно.
- Пользователь напомнил про награды и достижения — добавил замены в инвентарь, достижения и лавку.

## CSS-класс masonry-img (styles.css)
```css
.masonry-img {
  height: 100%;
  width: 100%;
  object-fit: contain;
}
```
