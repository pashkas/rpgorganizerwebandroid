# 0003 — Убрать вероятностное повышение уровня монстра в GetRndEnamy

## Запрос пользователя
«Картинки монстров EnamyImage рассчитываются от ранга персонажа + есть вероятность что выскочит картинка монстра следующего уровня. Убери это, пусть только в соответствии с рангом перса.»

## Проблема
В методе `GetRndEnamy` (pers.service.ts) после определения уровня монстра по рангу персонажа (`getMonsterLevel`) был блок кода, который:
1. Вычислял `nextBoundary` — ближайший уровень персонажа, на котором уровень монстра повышается
2. Вычислял `prevBoundary` — нижнюю границу текущего диапазона
3. Рассчитывал `progress` — долю прохождения текущего диапазона
4. С вероятностью `progress` повышал `mnstrLvl` на 1 (если < 6)

Это приводило к тому, что чем ближе персонаж к следующему рангу, тем чаще появлялись монстры следующего уровня.

## Решение
Удалён весь блок вычисления границ и вероятностного повышения (строки 174-197). Теперь `GetRndEnamy` просто берёт `mnstrLvl` из `getMonsterLevel(lvl, maxlvl)` и назначает картинку строго по этому уровню.
