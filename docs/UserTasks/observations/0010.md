# 0010. Потолок прокачки навыка в `recountAbilMayUp` Шаг 2 синхронизирован с `EraSettings.maxAbilLvl` только вручную

## Суть
Максимальное значение навыка, до которого разрешена прокачка по текущему уровню перса, живёт в `pers.service.ts` `recountAbilMayUp` Шаг 2 отдельной формулой (`max = ... floor(prs.level / 10) ...`). Эта формула **не вычисляется** от `this.gameSettings.maxAbilLvl` — константа в `EraSettings` задаёт только глобальный потолок (и потолок перков), а ступени навыков зашиты в `pers.service` отдельно. Смена `maxAbilLvl` в `EraSettings` без синхронной правки формулы в Шаге 2 оставит навыки с прежним потолком — задачи 0019 (maxAbilLvl=10 → `2 + floor/10 * 2`), 0025 (maxAbilLvl=5 → `1 + floor/10`) и 0026 (обратно к 10) каждый раз правили обе точки.

## Где в коде
- `src/app/game-settings/EraSettings.ts` — `maxAbilLvl`, `maxChaLvl` — общие константы потолка.
- `src/app/pers.service.ts` — `recountAbilMayUp`, Шаг 2 (около строки 3636) — формула `max` для обычных навыков; для перков в этом же шаге потолок берётся из `this.gameSettings.maxAbilLvl`.

## Почему важно
При правке баланса (любая смена `maxAbilLvl`) надо проверять Шаг 2 `recountAbilMayUp` и переподбирать формулу под диапазон уровней перса (1..`maxPersLevel`, сейчас 50) так, чтобы на максимальном уровне перса ступень достигала `maxAbilLvl`. Иначе `maxAbilLvl` ничего не значит для обычных навыков. Автоматической связки нет — только ручная синхронизация.
