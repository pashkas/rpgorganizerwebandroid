# 0012. Отображение ОП/ОН — три компонента, две разные схемы

## Суть

Пара валют ОП (star) / ОН (diamond) показывается в **трёх компонентах**, но с разными визуальными схемами:

1. **`pers-list` и `task-detail`** — bootstrap `badge badge-pill` в правом верхнем углу иконки. ОП = `badge-warning` (жёлтый), ОН = `badge-success` / `badge-danger` через `[ngClass]` по знаку. Число не стоит рядом с иконкой инлайн, а «приклеено» к её углу через `position: absolute; top: -4px; right: -4px`. Размеры бейджа — `font-size: 0.65em; padding: 2px 5px`.
2. **`level-up-msg`** — старая схема (до 0029): вертикальный стэк `star + число` сверху и `diamond + число` снизу, число — обычный `<span>` рядом с `<img>`, цвет текста через `color: gold` и `text-success`/`text-danger`. Высота иконок — `1.8em`, класс `.secImg` в `level-up-msg.component.css`.

Это **сознательная десинхронизация**: в 0029 пользователь попросил бейджи «на странице перса», `level-up-msg` явно оставили нетронутым.

## Где в коде

- [src/app/pers-list/pers-list.component.html](../../src/app/pers-list/pers-list.component.html) — главная вкладка: шапка с четырьмя иконками (опыт, gold, star, diamond) — все через badge-pill. Вкладки Инвентарь / Достижения / Лавка — шапка с gold через badge-pill.
- [src/app/task-detail/task-detail.component.html](../../src/app/task-detail/task-detail.component.html) — правая колонка ряда с картинкой навыка: star в правом верхнем углу, diamond по центру справа (через `flex: 1 1 auto`-спейсеры), кнопка up в правом нижнем. Левая колонка: кнопка down в левом нижнем. Иконки — фиксированные `height: 25px`.
- [src/app/level-up-msg/level-up-msg.component.html](../../src/app/level-up-msg/level-up-msg.component.html) — боковые блоки: вертикальные столбики `perkPoints + star` / `abPoints + diamond` с инлайн-числами. Класс `.secImg` задаёт `max-height: 1.8em`.

Условие показа ОП — `*ngIf="this.gameSettings.isAbPointsEnabled"`.

## Паттерн badge-pill на иконке

```
<span style="position: relative; display: inline-block; line-height: 0">
  <img src="icon.png" [ngStyle]="{'object-fit': 'contain', height: 'N'}" />
  <span class="badge badge-pill badge-X"
        style="position: absolute; top: -4px; right: -4px; font-size: 0.65em; padding: 2px 5px">
    {{ value }}
  </span>
</span>
```

Цвета: `badge-warning` — gold / ОП, `badge-success` — level / позитив, `badge-danger` — негатив (ON<0).

## Почему важно

- **При добавлении нового UI-места с балансом** (новая карточка, диалог, виджет) — уточнить у пользователя или определить по окружению: это «на странице перса» (→ badge-pill) или это уровень/попап (→ по канону `level-up-msg`, если переделывать единообразно).
- **При попытке «синхронизировать как в 0028»** — это устарело. 0029 разделил канон намеренно.
- **`pers.OP` может быть `undefined`** у старых персонажей (миграции нет, см. 0023). В `pers-list` при bage-pill выводится напрямую, отобразится пустой бейдж. В коде расчётов — подстраховывать через `?? 0`.
- **`pers.ON < 0`** — в `task-detail` и `pers-list` переключает бейдж в `badge-danger` через `[ngClass]`. В `level-up-msg` это ещё не применено (там в коде только неотрицательное ожидается).

## Обновление 2026-04-22

В задаче 0030 макет `level-up-msg` изменён: отказались от зеркальных столбиков `star + diamond` с двух сторон. Теперь слева — только `star + perkPoints` (ОП), справа — только `diamond + abPoints` (ОН). В правом блоке порядок внутри ряда инвертирован: сначала иконка, потом число.

Суть канона «в `level-up-msg` своя схема, отличная от `pers-list`/`task-detail`» сохраняется — просто форма канона стала другой: не стэк-зеркало, а разнесение валют по сторонам. Класс `.secImg` (height 1.8em), цвет `color: gold` для ОП, отсутствие badge-pill — всё на месте.
