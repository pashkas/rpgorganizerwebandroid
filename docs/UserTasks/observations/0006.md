# 0006. `GlobalItem` — view-model для masonry главного окна; визуальные флаги добавляем туда

## Суть
Плитки в masonry-сетке главного окна (и навыки, и квесты) отрисовываются не из `Task` напрямую, а из view-модели `GlobalItem`. Поля `GlobalItem` — проекции нужных для отображения атрибутов задачи (`name`, `img`, `tskId`, `progressValue`, `isNoActive`, `failCounter` и т.п.). Два метода `PersService` заполняют коллекции `skillsGlobal$` и `qwestsGlobal$`. Паттерн: новый визуальный признак на плитке (прозрачность, рамка, иконка, сортировка) = новое поле в `GlobalItem` + проброс в соответствующем `generateXxxGlobal`, шаблон читает поле из `GlobalItem`, а не лезет в `task` глубоко.

## Где в коде
- `src/Models/GlobalItem.ts` — класс (`name`, `img`, `tskId`, `tskIdx`, `task`, `progressValue`, `masonryIdx`, `isNoActive`, `failCounter`).
- `src/app/pers.service.ts:794` — `generateQwestsGlobal(prs)` — сборка и сортировка (неактивные в конец).
- `src/app/pers.service.ts:837` — `generateSkillsGlobal(prs)` — проставляет `gl.failCounter = tsk.failCounter || 0`.
- `src/app/pers.service.ts:2284-2285` — оба метода вызываются после `recalc`.
- `src/app/main-window/main-window.component.html` — шаблон читает `skill.failCounter`, `qwest.isNoActive` и т.п.

## Почему важно
Не стоит тянуть в шаблон главного окна поля прямо из `Task` через `gl.task.xxx` — это ломает паттерн и OnPush (полный объект задачи может мутироваться в другом месте). Всё, что должно повлиять на отрисовку плитки, проецируется отдельным полем `GlobalItem`. Сортировка/фильтрация «в конец» для неактивных тоже делается внутри `generateXxxGlobal`, не в шаблоне.
