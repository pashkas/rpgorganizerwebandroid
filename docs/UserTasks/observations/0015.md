# Скрытие последнего разделителя сегментов через `background-size`

## Суть

Когда делаешь сегментированную шкалу через `repeating-linear-gradient` с шагом 10% и 1px-разделителем, у тебя получается **10 разделителей**: 9 внутренних + 10-й сидит вплотную к правому краю элемента. Последний выглядит лишним — «незакрытый сегмент» / двойная линия рядом с правой границей.

Починка — НЕ перестраивать градиент, а раздуть background на 1px:

```css
.charactProgrSegments {
  background-image: repeating-linear-gradient(
    to right,
    transparent 0,
    transparent calc(10% - 1px),
    rgba(0,0,0,0.05) calc(10% - 1px),
    rgba(0,0,0,0.05) 10%
  );
  background-size: calc(100% + 1px) 100%;
}
```

`overflow: hidden` у контейнера клипнёт эту лишнюю 1px-полоску справа, и 10-й разделитель оказывается вне видимой области. Остаётся ровно 9 делителей между 10 сегментами.

## Где в коде

- [src/styles.css](../../src/styles.css) — класс `.charactProgrSegments`
- Используется в `pers-list`, `characteristic-details`, `task-detail` как overlay `<div class="charactProgrSegments">` с `position: absolute; width: 100%`.

## Почему важно

Простое визуальное раздражение, но альтернативы хуже:
- Изменить шаг градиента на `calc(100%/9)` — ломает ровные 10%-доли.
- Отдельный overlay, маскирующий последнюю 1px-полоску — лишний слой.
- 9 отдельных background-image с `linear-gradient` и `background-position` — читаемость на помойку.

Финт с `background-size: calc(100% + 1px)` — один дополнительный свойство, работает всегда, когда контейнер клипит overflow.
