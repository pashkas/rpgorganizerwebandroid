# Три разные страницы с похожими прогресс-барами

## Суть

В проекте три разных места с похожими по верстке прогресс-барами. Их легко перепутать по скриншоту, если не видно URL:

- **`pers-list`** — главная страница перса (`/pers/task-list` или корень `pers`), показывает список характеристик + навыков под ними. Прогресс-бар — у характеристики и у каждого навыка, обычно компактные.
- **`characteristic-details`** — страница **характеристики**, URL `/pers/characteristic/:id`. Большая картинка характеристики, один крупный прогресс-бар с именем характеристики, под ним — список навыков этой характеристики.
- **`task-detail`** — страница **задачи/навыка**, URL `/pers/task/:id`. Картинка навыка/задачи, один прогресс-бар с именем навыка.

## Где в коде

- `src/app/pers-list/pers-list.component.html` — главная страница перса
- `src/app/characteristic-details/characteristic-details.component.html` — окно характеристики
- `src/app/task-detail/task-detail.component.html` — окно задачи/навыка

## Почему важно

Пользователь часто присылает скрин без подписи «какая это страница». Если на скрине видна картинка-аватар + прогресс-бар + подпись типа «Навык» / «Характеристика» — всё равно стоит глянуть на URL (`/pers/task/...` vs `/pers/characteristic/...`) до того, как лезть в файлы. В этой сессии я дважды отредактировал `characteristic-details`, хотя пользователь демонстрировал `task-detail`, и потерял итерацию на повторное объяснение.

Эвристика: «окно **навыка**» в речи пользователя → `task-detail`. «Окно **характеристики**» → `characteristic-details`. «Окно перса» / «страница перса» → `pers-list`.
