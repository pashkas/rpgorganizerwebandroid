# 0004. Единая masonry-сетка — CSS Grid `.masonry-grid`, ngx-masonry и column-count удалены

## Суть
В проекте все плиточные сетки (навыки и квесты в main-window, квесты/инвентарь/достижения/лавка в pers-list, галерея картинок в add-item-dialog) раскладываются одним CSS-классом `.masonry-grid` на базе CSS Grid. Раскладка — по строкам (слева направо), адаптивность через media queries: 3 колонки мобилка / 5 средний / 8 большой экран. Модификатор `.masonry-grid.dense` переопределяет мобильный режим на 4 колонки (используется для навыков при `skills.length > 15`). Библиотека `ngx-masonry`, старые классы `masonry-list`, `masonry-list-galery`, `column-count` — удалены из кода (пакет остался в `package.json`, но не импортируется).

## Где в коде
- `src/styles.css:442` — `.masonry-grid`, `.masonry-grid.dense`, media queries для 3/4/5/8 колонок.
- `src/styles.css` — `.masonry-container` с `padding: 2px`, совпадающим с `gap: 2px` в `.masonry-grid`.
- `src/app/main-window/main-window.component.html` — навыки и квесты.
- `src/app/pers-list/pers-list.component.html` — квесты, инвентарь, достижения, лавка.
- `src/app/add-item-dialog/add-item-dialog.component.html` — галерея картинок.
- Удалённые файлы: `src/app/shared/masonry.pipe.ts`, `SharedModule` больше не экспортирует `NgxMasonryModule`.

## Почему важно
Новая сетка где-либо в UI — использовать `.masonry-grid` (и `.dense`, если плиток много). НЕ тянуть `ngx-masonry` обратно, НЕ использовать `column-count` (он раскладывает по столбцам, а в проекте инвариант — порядок по строкам совпадает с порядком данных). Картинки в плитках унифицированы классом `.masonry-img` (`height:100%; width:100%; object-fit:contain`) — используем его, не `img-fluid rounded w-100`.
