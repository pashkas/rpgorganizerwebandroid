# 0007. Ловушки Material 8 `mat-button-toggle`: `[(value)]` и `focus-overlay`

## Суть
Angular Material 8 (версия в проекте) имеет две неочевидные проблемы с `mat-button-toggle-group`:

1. **`[(value)]` работает нестабильно** при быстром переключении — подсветка выбранного может «отставать» или не обновляться. Рабочий паттерн: на каждом `mat-button-toggle` руками `[checked]="tsk.field === value"` + `(click)="tsk.field = value"`.
2. **`.mat-button-toggle-focus-overlay`** после клика оставляет белый слой поверх background выбранной кнопки — визуально «выбор пропадает». Обязательно делать прозрачным (`background: transparent !important`) через `::ng-deep`.

## Где в коде
- `src/app/task-detail/task-detail.component.html` — `<mat-button-toggle-group>` для сложности перка: `[checked]`+`(click)` вместо `[(value)]`.
- `src/app/task-detail/task-detail.component.css:33-44` — `::ng-deep .mat-button-toggle-checked` + `.mat-button-toggle-focus-overlay` стили для надёжной подсветки.
- `src/app/shared/shared.module.ts` — `MatButtonToggleModule` импортирован и экспортирован.

## Почему важно
Если в следующий раз понадобится любой переключатель «2-3 варианта» (сложность/режим/тип) — не тянуть `mat-button-toggle-group` с `[(value)]`, сразу писать через `[checked]`+`(click)` и не забывать `focus-overlay: transparent` в CSS. Иначе будут жалобы вида «кнопка выбирается через раз» или «выделение пропадает после клика», и придётся переделывать. Обновлять Material до версии, где `[(value)]` надёжен, не рассматриваем — проект жёстко на Material 8.
