# 0005. OnPush + мутации коллекций — пересоздавать ссылку и звать `markForCheck()`

## Суть
Большинство компонентов в `PersModule` живут с `ChangeDetectionStrategy.OnPush`. В таком режиме Angular не видит in-place мутаций: `set.add(x)`, `arr.push(x)`, `obj.foo = true` не триггерят CD, пока ссылка не поменялась. Плюс динамические свойства на элементах массива теряются при каждом пересоздании массива из `BehaviorSubject`. Рабочий паттерн: обработчик события создаёт новый экземпляр коллекции (или хранит индикатор в `Set`/`Map` на самом компоненте), затем явно зовёт `cdr.markForCheck()`.

## Где в коде
- `src/app/main-window/main-window.component.ts:35` — `loadedSkillImgs = new Set<string>()` хранится на компоненте, а не на элементах `skillsGlobal$`.
- `src/app/main-window/main-window.component.ts:325-328` — `onSkillImgLoad(tskId)`: `this.loadedSkillImgs = new Set(this.loadedSkillImgs).add(tskId); this.cdr.markForCheck();` — сначала новая ссылка, потом CD.
- `src/app/main-window/main-window.component.html` — `(load)="onSkillImgLoad(skill.tskId)"`, условие в `[ngStyle]` читает из Set на компоненте.

## Почему важно
При добавлении любого UI-флага, который выставляется по событию `(load)`/`(click)`/подписке на Observable в OnPush-компоненте: никаких `obj.flag = true` на элементе массива, пришедшего из сервиса — будет потеря при следующем `next()` и отсутствие CD. Храним состояние на уровне компонента в Set/Map, при обновлении пересоздаём ссылку и зовём `markForCheck()`. Иначе визуал либо не обновится вовсе, либо будет мигать в зависимости от порядка CD-циклов.
