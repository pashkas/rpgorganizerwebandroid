# 0005 — Улучшение директивы свайпов CustomSwipeDirective

## Запрос пользователя
«В главном окне используются свайпы через директиву. Можешь как-то улучшить эту директиву?»

## Цель
Улучшить качество кода и UX директивы `CustomSwipeDirective`, которая обрабатывает свайпы в main-window (переключение видов, done/fail по задаче).

## Проблемы до рефакторинга
- Минимальная дистанция свайпа 30px — слишком мала, ложные срабатывания
- Нет блокировки скролла при горизонтальном свайпе (touchmove не перехватывался)
- Лишняя промежуточная обёртка `onSwipe()` → `swipe()`
- Публичные поля `swipeCoord`, `swipeTime`
- Магические числа без возможности настройки

## Что сделано
1. **Настраиваемые пороги через `@Input()`:**
   - `swipeThreshold` = 50px (было 30) — минимальная дистанция
   - `swipeMaxDuration` = 800мс (было 1000) — максимальная длительность
   - `swipeDirectionRatio` = 3 — коэффициент перевеса направления
2. **Добавлен `touchmove` listener** — при распознании горизонтального свайпа вызывается `preventDefault()`, блокируя скролл страницы
3. **Рефакторинг структуры:** убрана обёртка, приватизированы поля, добавлен `reset()`
4. **API не менялся** — селектор `swipe`, выходы `next`, `previous`, `up`, `down` — всё как было

## Затронутые файлы
- `src/app/customSwipe.directive.ts` — полностью переписана

## Побочные эффекты
- Порог свайпа увеличен с 30px до 50px — короткие случайные жесты больше не будут срабатывать
- Максимальная длительность уменьшена с 1000мс до 800мс — очень медленные свайпы игнорируются
- Оба порога можно переопределить через `@Input()` в шаблоне при необходимости
