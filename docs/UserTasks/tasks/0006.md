# 0006 — Красная рамка для пропущенных задач в masonry навыков

## Запрос пользователя
«Если задача раньше была пропущена (failCounter > 0), в masonry навыков выделять тонкой красной рамкой»

## Цель
Визуально отличать в masonry-сетке навыков те плитки, задача которых была ранее пропущена (fail), чтобы пользователь сразу видел «проблемные» навыки.

## Подход
- В `GlobalItem` добавлено поле `failCounter` — хранит значение `failCounter` из связанной задачи.
- В `PersService.generateSkillsGlobal()` при создании `GlobalItem` заполняется `gl.failCounter = tsk.failCounter || 0`.
- В шаблоне `main-window.component.html` на элементе `ngxMasonryItem` добавлен `[ngStyle]` с условием: если `failCounter > 0` — рисуется `1px solid red` рамка с `border-radius: 4px`.

## Затронутые файлы
- `src/Models/GlobalItem.ts` — новое поле `failCounter`
- `src/app/pers.service.ts` — заполнение `failCounter` в `generateSkillsGlobal()`
- `src/app/main-window/main-window.component.html` — условная красная рамка в masonry навыков

## Побочные эффекты
- Нет. Поле `failCounter` опционально (`|| 0`), существующие данные не ломаются.
