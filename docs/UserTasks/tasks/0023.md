# 0023 — Очки перков (ОП) как отдельная валюта

## Запрос пользователя

> «Сейчас есть очки навыков (ОН, diamonds). Очки навыков получают за каждый уровень (5 штук). Я хочу сделать "Очки перков". Получаем 1 очко перков за каждый третий уровень персонажа, а очки навыков на каждом третьем уровне персонажа не получаем. Визуально это будет выглядеть как зеленое число в зеленых скобочках после числа очков навыков. Например "5 (1)". Навыки можно будет покупать на очки навыков. Перки - только на очки перков. Норм перк будет стоить 1 очко, сложн 2 очка. Также убери логику, что перки можно прокачивать только когда нет доступных для прокачки навыков и что перк надо "докачивать".»

## Цель

Разделить экономику прокачки: обычные навыки покупаются за ОН, перки — за новый ресурс ОП. Каждый 3-й уровень персонажа «отнимается» у ОН и начисляет 1 ОП. Попутно убрать оба ограничения из 0022 — «перки только при залоченной базе» и «докачка сложн-перка».

## Экономика

- **Начисление:** gained_ON = `(persLevel - floor(persLevel/3)) * abPointsPerLvl + abPointsStart`. gained_OP = `floor(persLevel/3)`. Порог `perkPointLvlInterval` = 3 живёт в `GameSettings` и может меняться режимом.
- **Цена:** норм-перк = 1 ОП (один шаг до `maxAbilLvl`), сложн-перк = 2 ОП (два шага по 1 ОП: value 0→5→10).
- **«Докачка» сложн-перка:** сложн-перк можно начать с 1 ОП (полкачки, value=0→5). После этого ВСЁ остальное в персонаже блокируется на прокачку, пока этот сложн не добьют до maxAbilLvl. Этот ре-введённый Шаг 3 работает только для сложн с 0<value<maxAbilLvl (норм такого состояния не имеет — он сразу до maxAbilLvl).

## Подход

### Модель
- [src/Models/Pers.ts](../../src/Models/Pers.ts) — поле `OP: number = 0`.

### Игровые настройки
- [src/app/GameSettings.ts](../../src/app/GameSettings.ts):
  - `perkPointLvlInterval: number = 3`.
  - `perkCost(value, perkHardnes)` — пороговая стоимость ОП для `mayUp`. Сложн при value=0 = 2, иначе 1.
  - `perkTotalCost(value, perkHardnes)` — сумма потраченных ОП на перк (для расчёта баланса): норм = 1 если value>0, сложн = 1 при 0<value≤5, 2 при value>5.
- [src/app/game-settings/EraSettings.ts](../../src/app/game-settings/EraSettings.ts):
  - `abCost` и `abTotalCost` для перков возвращают **0** — перки больше не расходуют пул ОН, только ОП.

### Сервис персонажа
- [src/app/pers.service.ts](../../src/app/pers.service.ts):
  - `getClassicalAbPoints`: из общего числа уровней вычитается `floor(persLevel/perkPointLvlInterval)` перед умножением на `abPointsPerLvl`.
  - Новый `getClassicalPerkPoints(persLevel, opValTotal)` — начислено минус потрачено.
  - В `recalc` заведена локальная `opValTotal`, накапливается `perkTotalCost(tsk.value, tsk.perkHardnes)` для перков, `prs.OP` присваивается после расчёта уровня.
  - `recountAbilMayUp` полностью переписан:
    - Шаг 1: раздельные проверки — для перков `op < perkCost(tsk.value, tsk.perkHardnes)`, для обычных `on < abCost(...)`.
    - Шаг 2: потолок по уровню перса (из 0019) — без изменений.
    - Шаги 2 и 4 из задачи 0022 удалены — `anySame` / `IsNextLvlSame` / «докачка сложн-перка» / блок «перки только если нет обычных» вырезаны целиком. `HasSameAbLvl` сбрасывается в false (поле оставлено, но не используется — если всплывёт в шаблонах, там просто не сработают условия).

### UI

Формат отображения во всех местах: `5 / 1` — ОН зелёный (всегда, даже 0), ОП золотой (всегда, даже 0), разделитель «/».

- [src/app/task-detail/task-detail.component.html](../../src/app/task-detail/task-detail.component.html) — правая колонка карточки. Переписана целиком: убран `matBadge`, контейнер с `flex: 0 0 50px` расширен до `auto` и `white-space: nowrap`, т.к. «(1)» переносилось на новую строку.
- [src/app/characteristic-details/characteristic-details.component.html](../../src/app/characteristic-details/characteristic-details.component.html) — условие показа кнопки прокачки у ability: для перков проверяется `pers.OP > 0`, для обычных `pers.ON > 0`.
- [src/app/level-up-msg/level-up-msg.component.html](../../src/app/level-up-msg/level-up-msg.component.html) + [.ts](../../src/app/level-up-msg/level-up-msg.component.ts) — добавлено поле `perkPoints`, формат `5 / 1` рядом с алмазом.
- [src/app/perschanges.service.ts](../../src/app/perschanges.service.ts) — `this.afterPers.OP` прокидывается в `LevelUpMsgComponent` как `perkPoints`.

### Цвет
- ОН: `text-success` (зелёный), `text-danger` когда уходит в минус.
- ОП: inline `style="color: gold"`. Bootstrap-класса под gold в проекте нет, завели inline — на случай локального оформления можно вынести в `styles.css`.

## Итерации UX

Обсудили формат отображения — перебрали «5 (1)», «0 - 1», звёздочку. Остановились на «5 / 1» с ОН зелёным и ОП золотым; показываем оба всегда, даже при 0 (пользователь явно попросил). В карточке навыка первоначально было пропало окрашивание ОН — восстановили `text-success` при любом неотрицательном значении.

## Побочные эффекты / edge cases

- **Миграция старых персонажей:** у `pers.OP` нет миграции — при загрузке старого перса `OP = undefined`. `recountAbilMayUp` подстраховывается `op = prs.OP ?? 0`, а при первом `recalc` поле корректно заполняется из `getClassicalPerkPoints`. Персы с уже прокачанными перками до этой задачи будут считать `opValTotal` ретроспективно — возможно отрицательное `OP`, если до изменений на перки было потрачено больше ОП, чем накоплено по формуле. На практике: перки на высоких уровнях уже оплачены прежней системой, и пользователь стартует «в долг». Если всплывёт — отдельная таска на одноразовый пересчёт.
- **`HasSameAbLvl` / `IsNextLvlSame`:** поля не удалены из `Task` и сбрасываются в false. Если где-то в шаблонах остались условия на них — визуально перки просто не получат спец-отрисовку «докачка». Это ожидаемо, т.к. механика докачки удалена.
- **`characteristic-details`:** кнопка прокачки для перка теперь скрывается, если ОП=0. Раньше скрывалась, если ОН=0 — то есть перки разблокировались только когда ОН был. Теперь перки живут своей жизнью.

## Корректировки от пользователя

1. В интерфейсе «5 (1)» в узкой колонке карточки переносилось на строку ниже — переделал на inline без `matBadge`, расширил контейнер.
2. ОП изначально был зелёным — попросил сделать золотым, чтобы отличать от ОН.
3. ОН пропало окрашивание в карточке навыка — вернул `text-success` при значении ≥ 0.
4. Рассмотрели варианты отображения («5 (1)» / «5 - 1» / иконка звезды / «5 / 1») — выбрали slash.
5. Сначала скрывали ОП при 0 — переделали на «всегда показывать», включая 0 для ОН и ОП.
6. Обсудили поведение сложн-перка при 1 ОП. Сначала сделали гейтинг через `perkCost` (2 ОП upfront), потом пользователь передумал — вернули возможность начать с 1 ОП, но с блокировкой всего остального до «докачки» (перенос Шага 2 из 0022 на новый пул ОП). `perkCost` возвращена к константе 1.
