# Прогресс-бары в попапе изменений — привести к современному стилю

## Запрос пользователя

> «сделай в изменениях (changesItem) прогресс бары как в других местах»

## Подход

Применили паттерн из задачи 0032 (`.abProgr` + сегменты + градиентная заливка) к `pers-changes-item`. Выбран бесрамочный `.abProgr` — попап плотный, рамка на каждой строке визуально перегружала бы. Заливка — через геттер `fillClass`:

- `cha`, `exp` → `.charactProgrFill` (зелёный градиент);
- `abil` → `.skillProgrFill` (жёлтый градиент);
- `hp` и прочие — пустой класс, инлайн `background-color: item.color` остаётся (для `hp` это `palevioletred`, задаётся в `ChangesModel`).

Текст строки (`{{item.name}}{{item.valChange+plusName}}`) переехал на `z-index: 3` поверх оверлея сегментов (`z-index: 2`) и заливки (`z-index: 1`). Высота 24px — как у строк навыков на `pers-list`. Анимация ширины через `AnimationBuilder` и `@ViewChild('progress')` сохранена без изменений — ViewChild всё так же указывает на элемент с классом `progress-bar`.

## Затронуто

- [src/app/pers-changes-item/pers-changes-item.component.html](../../src/app/pers-changes-item/pers-changes-item.component.html) — переход на паттерн `.abProgr` + `.charactProgrSegments` + текстовый оверлей.
- [src/app/pers-changes-item/pers-changes-item.component.ts](../../src/app/pers-changes-item/pers-changes-item.component.ts) — добавлен геттер `fillClass` по `item.type`.
