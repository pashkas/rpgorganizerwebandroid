# 0018 — Замена формулы коэффициента `e` в getPersExpAndLevel на сигмоиду с плавным ростом

## Запрос пользователя
«Хочу поменять формулу e в getPersExpAndLevel. На persLevel=1 должно быть ровно 1, макс 5. Слабый рост до 10, основной рост 10–20, после 20 слабый рост. Рост 10–20 до 3, округлять до десятых вниз.»

## Проблема
Старая формула `e = 2 + Math.floor(persLevel / 10)` давала линейный ступенчатый рост и не позволяла тонко настроить кривую сложности по диапазонам уровней.

## Решение
Заменили на сигмоиду с параметрами, подобранными под требования:

```ts
let e = Math.floor((1 + 4 / (1 + Math.exp(-0.35 * (persLevel - 18)))) * 10) / 10;
```

### Параметры сигмоиды
- Базовое значение: 1 (при уровне 1)
- Амплитуда: 4 (макс = 1 + 4 = 5)
- Крутизна (k): 0.35
- Точка перегиба (mid): 18

### Итоговые значения e по уровням
| Уровень | e   |
|---------|-----|
| 1       | 1.0 |
| 10      | 1.2 |
| 20      | 2.9 |
| 30      | 4.7 |
| 40      | 4.9 |
| 50      | 5.0 |

### Округление
`Math.floor(... * 10) / 10` — округление вниз до десятых, по просьбе пользователя.

## Дополнительно
Добавлены комментарии к циклу определения уровня персонажа: назначение переменных, суть коэффициента, условие выхода.

## Затронутые файлы
- `src/app/game-settings/EraSettings.ts` — метод `getPersExpAndLevel`, формула `e` + комментарии

## Корректировки от пользователя
1. Первый вариант сигмоиды (k=0.3, mid=15) давал слишком сильный рост 10–20 (~1→3 = +2). Пользователь попросил уменьшить до 3 к 20-му уровню.
2. Подобран вариант с k=0.35, mid=18 — рост 10–20 = +1.7, на 20-м уровне e≈2.9.
3. Попросил округлять до десятых вниз — добавлен `Math.floor(... * 10) / 10`.
4. Попросил прокомментировать код — добавлены комментарии.
