# 0024 — Интервал начисления ОП: 3 → 4 уровня

## Запрос пользователя

> «А где тут расчитывается что один оп за каждый третий уровень?» → «Поменяй на каждый четвертый.»

## Цель

Снизить темп получения очков перков (ОП): теперь 1 ОП начисляется за каждый 4-й уровень персонажа вместо каждого 3-го. Соответственно каждый 4-й уровень не даёт очков навыков (ОН) — он «отнимается» в пользу ОП.

## Подход

Изменена единственная константа `perkPointLvlInterval` в [src/app/GameSettings.ts:165](../../src/app/GameSettings.ts#L165) — с `3` на `4`. Вся остальная логика из 0023 продолжает работать без изменений:
- `getClassicalPerkPoints` ([pers.service.ts:953-955](../../src/app/pers.service.ts#L953-L955)) — `Math.floor(persLevel / perkPointLvlInterval)` минус потраченные ОП.
- `getClassicalAbPoints` ([pers.service.ts:940-942](../../src/app/pers.service.ts#L940-L942)) — те же `perkLvls = Math.floor(persLevel / perkPointLvlInterval)` вычитаются из общих уровней перед умножением на `abPointsPerLvl`.

## Побочные эффекты

- Старые персонажи на пересчёте получат меньше ОП и больше ОН, чем при интервале 3. Например, на 12-м уровне раньше было 4 ОП, теперь — 3 ОП. ОН на ту же сумму ×`abPointsPerLvl` больше.
- Если у перса уже потрачено ОП больше, чем даёт новая формула, `prs.OP` уйдёт в минус — поведение `recountAbilMayUp` под это рассчитано (`op = prs.OP ?? 0`, проверка `op < perkCost(...)`).
