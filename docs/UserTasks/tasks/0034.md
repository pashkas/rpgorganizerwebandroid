# Активация навыка 0→1 показывается прогресс-баром вместо текста «активирован»

## Запрос пользователя
«Когда поднимаем навык с 0 до 1 в changeService пишет "активирован". Сделай чтобы просто показывал прогресс бар с движением от 0 до 1, как при следующих значениях (с 2 до 3, например).»

Следом уточнил: «Не вижу движение прогресс бара с 0 до 1 в changesItem. Там наверное начало отсчета с 1».

## Подход
1. В `PerschangesService.showChanges` ветка «Активирован» (`el.abIsOpenBefore != el.abIsOpenAfter`) теперь срабатывает только для перков. Для обычных навыков 0→1 проваливается в стандартную ветку `changesIsShowAbLevels` и формирует обычный `ChangesModel` уровня.
2. В этой же ветке — при `el.before < minAbilLvl` (т.е. активация) в `ChangesModel` передаём `totalMin = 0` вместо `minAbilLvl=1`. Иначе нормализация `valFrom = (0-1)/(5-1)*100 = -25%` и бар «анимируется» от -25% до 0% — визуально ничего не видно. С `totalMin=0` бар идёт 0% → 20%, как обычный шаг уровня.

Для перков поведение сохранено — «перк активирован»/«перк прокачан» по-прежнему показывается (см. задачу 0015).

## Затронуто
- Файлы: `src/app/perschanges.service.ts`
- Методы: `PerschangesService.showChanges` — ветки `el.type == "abil"` (активация и изменение уровня)
