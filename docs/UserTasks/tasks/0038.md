# Прогресс-бар в changes-popup: разделители, шаг 1→0 и точка отсчёта характеристик

## Запрос пользователя
«При изменениях навыков (может характеристик) в changeItem, changeService надо поправить. Когда поднимается с 0 до 1 все норм. А когда с 1 до 2 как будто прогресс бар движется с нуля и разделители поехали.»
«При "опускании" навыка с 1 до 0 хочу чтоб тоже показывал прогресс бар.»
«Проследи чтобы в характеристиках "точка отсчета" начиналась также с 0 как в навыках, а не с MinLevel.»
«Только полоса опыта должна быть желтой, как в mainWindow.»
«Когда меняется "опыт" или "квесты" в changesItem не показывать разделители? Разделители только для характеристик и навыков.»

## Подход
1. В `PerschangesService.showChanges` для навыков (ветка level) `totalMin = 0` вместо `minAbilLvl`. Аналогично для характеристик (ветка cha level) — `totalMin = 0` вместо `minChaLvl`. Костыль `abTotalMin = el.before < minAbilLvl ? 0 : minAbilLvl` из задачи 0034 больше не нужен — теперь всё единообразно. Побочный бонус: шаг уровня = 10% при `maxAbilLvl=10` и `maxChaLvl=10`, что совпадает с 10%-ми `.charactProgrSegments` — разделители перестают «ехать». Заодно 1→0 анимируется 10%→0% (раньше `valTo=(0-1)/9*100` клампился в 0% и бар стоял).
2. В `pers-changes-item.component.ts` `fillClass` для `exp` переключён с `charactProgrFill` (зелёный) на `skillProgrFill` (жёлтый) — как в mainWindow.
3. В `pers-changes-item.component.html` разделители `.charactProgrSegments` обёрнуты в `*ngIf="item.type == 'cha' || item.type == 'abil'"`. Для `exp`, `qwest`, `inv`, `gold`, `hp` разделителей больше нет.

## Затронуто
- `src/app/perschanges.service.ts` — `showChanges` (ветки `cha` level и `abil` level)
- `src/app/pers-changes-item/pers-changes-item.component.ts` — `fillClass`
- `src/app/pers-changes-item/pers-changes-item.component.html` — `.charactProgrSegments` обёрнут в `*ngIf`
