# 0017 — Замена линейной кривой опыта на S-кривую (сигмоид)

## Запрос пользователя

> Обсудить и переделать формулу расчёта `e` (множитель длительности уровня) в `EraSettings.getPersExpAndLevel`. Текущая линейная формула `e = 1 + (persLevel - 1) * 0.05` не учитывает психологию адаптации к привычкам — после 20-го уровня пользователь не успевает морально адаптироваться и «проскакивает» в выгорание.

## Проблема

Проект — геймифицированный таск-менеджер (аналог Habitica). Формула опыта состоит из двух частей:
- `5 * persLevel` — объём работы (количество/сложность задач), растёт с прокачкой навыков. **Эта часть не менялась.**
- `e` — множитель длительности, определяющий сколько «дней» нужно на уровень. **Эту часть меняли.**

Старая формула `e = 1 + (persLevel - 1) * 0.05` давала линейный рост 5% за уровень. Проблемы:
1. **До 10-го уровня** — энтузиазм высокий, но прогресс ощущается медленным (кураж и дофамин)
2. **10–20** — нормально, пользователь втягивается
3. **После 20-го** — сложность продолжает расти линейно, пользователь не успевает адаптироваться → выгорание. «Фундамент» не построен.

## Рассмотренные варианты

### 1. Асимптота
`e = 1 + maxE * (persLevel - 1) / (persLevel - 1 + C)`

- Плюс: рост убывает, есть потолок
- Минус: **сразу давит с первых уровней** (на 5-м уровне e уже ~2.14), нет «лёгкого» периода на энтузиазме. Не подходит для поддержки начального мотивационного всплеска.

### 2. Линейная, масштабированная к 5
`e = 1 + (persLevel - 1) * 4/49`

- Плюс: простая, достигает 5 на 50-м уровне
- Минус: равномерный рост без учёта фаз адаптации. Нагружает одинаково на всех этапах.

### 3. S-кривая (сигмоид) ← ВЫБРАНО
`e = 1 + 4 * (sig(level) - sig(1)) / (1 - sig(1))`, где `sig(x) = 1 / (1 + exp(-0.2 * (x - 22)))`

- e(1) = 1 (целое число на старте)
- До 10-го уровня — почти не растёт (1.00 → 1.28), поддерживает энтузиазм
- 10–30 — основной рост, фаза «строительства фундамента»
- После 30 — плато, стремится к 5
- Психологически соответствует циклу: **энтузиазм → рутина/адаптация → стабильность**

## Решение

Заменена формула `e` в методе `EraSettings.getPersExpAndLevel`:

**Было:**
```typescript
let e = 1 + (persLevel - 1) * 0.05;
```

**Стало:**
```typescript
let sig = 1 / (1 + Math.exp(-0.2 * (persLevel - 22)));
let sig1 = 1 / (1 + Math.exp(-0.2 * (1 - 22)));
let e = 1 + 4 * (sig - sig1) / (1 - sig1);
```

Параметры S-кривой:
- `k = 0.2` — крутизна перехода
- `mid = 22` — точка перегиба (середина фазы «фундамента»)
- `maxE = 4` — максимальная надбавка (e стремится к 5)
- Нормализация через `sig1` гарантирует e(1) = 1

## Затронутые файлы

- `src/app/game-settings/EraSettings.ts` — метод `getPersExpAndLevel`, строка с формулой `e`

## Побочные эффекты

- Все остальные части формулы (`abPointsPerLvl * persLevel * e`) не изменились
- Метод `changeExpClassical` (начисление опыта за задачи) не затронут
- Для персонажей выше 20-го уровня **пороги уровней изменятся** — текущий уровень может пересчитаться (скорее всего в большую сторону, т.к. S-кривая до 20-го мягче старой)
- Для персонажей ниже 15-го уровня — разница минимальна

## Корректировки от пользователя

- Изначально рассматривалась асимптота — отклонена из-за слишком агрессивного старта
- Пользователь уточнил требование: «до 10-го уровня поддержать энтузиазм, с 10 по 30 — критический период строительства фундамента»
- Требование: e(1) = целое число (1)
- Требование: e стремится к 5
- Итоговый выбор — S-кривая как лучше всего отражающая цикл «энтузиазм → адаптация → стабильность»
