# 0015 — Настраиваемая сложность перков (норм / сложн)

## Запрос пользователя

> «Навыки могут быть перками (isPerk). В GameSettings прописано что стоимость прокачки навыка=10 очков. Я хочу сделать возможность настраивать для перков сложность (только для перков). По умолчанию сложность должна быть "норм", тогда стоимость прокачки=5. И можно выбрать "сложн", тогда будет 10. Я хочу чтобы это выглядело как список, в котором две горизонтальных кнопки, выбранный уровень сложности подсвечивает нужную кнопку. Имей ввиду, что от сложности перка зависит сколько за него давать опыта (для норм должно быть в 2 раза меньше)»

## Цель

У перка (`Task.isPerk`) добавить явный выбор сложности: **норм** (дешевле, но и опыта меньше) или **сложн** (дороже и полноценный опыт). Раньше все перки вели себя одинаково: прокачивались в два шага по 5 ОН (итого 10 ОН), опыт = `curLvl * hardness` независимо от `isPerk`.

## Что получилось

- **норм** (`perkHardnes = 0.5`, по умолчанию): первый апгрейд сразу делает `tsk.value = maxAbilLvl` (10), списывается **5 ОН**, опыт за задачу — `×0.5`.
- **сложн** (`perkHardnes = 1`): сохраняется старая двухшаговая логика — первый апгрейд даёт `value=5`, второй `value=10`, итого 10 ОН, опыт `×1`.

В шапке задачи (через `AbHardnessPipe`) у перка «сложн» рядом с названием добавляется `*` (визуальный маркер сложности, как и у обычной задачи с `hardnes=2`).

В попапе изменений:
- обычный навык → `<имя> активирован`;
- перк «сложн» → `<имя> — перк активирован` (после первого апгрейда, половина);
- перк «норм» → `<имя> — перк прокачан` (сразу полный).

## Подход

### Модель
- [src/Models/Task.ts](../../src/Models/Task.ts) — новое поле `perkHardnes: number = 0.5` (число 0.5 / 1 — выбрано вместо enum-строки, чтобы прямо использовать как множитель в формулах опыта/цены).

### Игровая логика
- [src/app/GameSettings.ts](../../src/app/GameSettings.ts) — сигнатуры `abChangeExp` и `abTotalCost` получили опциональный параметр `perkHardnes?: number` (прочие реализации `TesSettings*`/`EraSettingsV*` просто его игнорируют).
- [src/app/game-settings/EraSettings.ts](../../src/app/game-settings/EraSettings.ts):
  - `abChangeExp`: `curLvl * hardness * (isPerk ? perkHardnes : 1)` — норм получает половину опыта.
  - `abTotalCost`: `curLvl * hardness * (isPerk ? perkHardnes : 1)` — ОН-стоимость полностью прокачанного перка: 5 для норм, 10 для сложн.
  - `checkPerkTskValue`: добавлена инициализация `tsk.perkHardnes = 0.5`, если `null`. Снап значения (5/10) нужен для сложн; для норм значение уже ставится `maxAbilLvl` в `upAbility`.

### Сервис персонажа
- [src/app/pers.service.ts](../../src/app/pers.service.ts):
  - `upAbility`: для перка «норм» сразу `tsk.value = gameSettings.maxAbilLvl` (один шаг). Сложн — обычный `+=1` + снап в `checkPerkTskValue`.
  - Пробросы `tsk.perkHardnes` в вызовы `abChangeExp` (строки ~2199, ~2990) и `abTotalCost` (~1887).
  - Guard в `recalc`: для перка `mayUp = false`, если `value >= maxAbilLvl` (чтобы норм не «докачивался» повторно).

### UI
- [src/app/shared/shared.module.ts](../../src/app/shared/shared.module.ts) — импорт+экспорт `MatButtonToggleModule` (раньше нигде не использовался).
- [src/app/task-detail/task-detail.component.html](../../src/app/task-detail/task-detail.component.html) — под чипом «Перк?» появляется `<mat-button-toggle-group>` с кнопками «норм»/«сложн» на всю ширину. Используется `[checked]` + `(click)` вместо `[(value)]`, потому что с двусторонним `value` подсветка периодически не обновлялась при быстром переключении.
- [src/app/task-detail/task-detail.component.ts](../../src/app/task-detail/task-detail.component.ts) — в `findTask` добавлена инициализация `tsk.perkHardnes ??= 0.5` для задач из БД без поля.
- [src/app/task-detail/task-detail.component.css](../../src/app/task-detail/task-detail.component.css) — CSS:
  - `width: 100%` + `flex: 1` — на всю ширину, по 50%.
  - компактный размер (`line-height: 30px`, `font-size: 13px`).
  - явная подсветка выбранной кнопки: `.mat-button-toggle-checked { background: #3f51b5; color: #fff; }` и `.mat-button-toggle-focus-overlay: transparent` — иначе после клика фокус-overlay закрашивал фон белым и выбор «пропадал».

### Визуальный маркер
- [src/app/ab-hardness.pipe.ts](../../src/app/ab-hardness.pipe.ts) — для `isPerk && perkHardnes === 1` к имени добавляется `*` (раньше в этой ветке было закомментировано `^`).

### Попап изменений
- [src/app/perschanges.service.ts](../../src/app/perschanges.service.ts):
  - В `changesMap[tsk.id]` при сборе состояния добавлены `isPerk` и `perkHardnes`.
  - Ветка «навык активирован» разветвлена: для перков — «перк активирован» (сложн) или «перк прокачан» (норм).
  - Поля `isPerk`, `perkHardnes` добавлены в класс `PersChangesItem` внизу файла.

## Побочные эффекты / edge cases

- **Миграция старых перков**: задачи из БД без `perkHardnes` подхватываются как `0.5` (норм) через fallback в `findTask` и `checkPerkTskValue`. Если старый перк был «на полпути» (`value=5`) — при следующем `checkPerkTskValue` сложн корректно держит 5, норм снапится до `maxAbilLvl` (докачается автоматически).
- **Подсветка `mat-button-toggle`**: Material 8 имеет тонкую проблему — `.mat-button-toggle-focus-overlay` после клика оставляет белый слой поверх background, из-за чего визуально «выбор пропадает». Обязательно делать overlay прозрачным.
- **`[(value)]` на `mat-button-toggle-group`** в Material 8 работает нестабильно при быстром переключении — надёжнее `[checked]` + `(click)` на каждом toggle.

## Корректировки от пользователя

1. Изначально планировалось хранить сложность как строку `'norm'|'hard'` — пользователь попросил просто числом (0.5 / 1), чтобы сразу использовать как множитель.
2. Первый вариант UI был `mat-chip-list` — заменён на `mat-button-toggle-group` по просьбе пользователя.
3. Кнопки изначально стояли «рядом с Перк?» — перенесено на отдельную строку под чипом.
4. Кнопки нужно было растянуть на всю ширину и сделать компактнее по высоте.
5. Подсветка выбранной работала через раз — переделал на `[checked]` + `(click)` и подкрутил CSS (focus-overlay, !important на checked background).
6. Забыли про `abTotalCost`: после прокачки норма ОН уходили в минус, т.к. total-cost возвращал полные 10. Добавил `perkHardnes` и туда.
7. Пользователь попросил, чтобы при прокачке перка в попапе писало «перк активирован» (сложн) / «перк прокачан» (норм) вместо «навык активирован».
8. Пользователь попросил визуальный маркер `*` у перка «сложн» — добавил в `AbHardnessPipe`.
