# 0021 — Квест с непрокачанным навыком (value=0) должен показываться как неактивный

## Запрос пользователя
«Тут у квестов есть связь с навыками. Они "серенькие" и в конце если задачи навыка в данный момент неактивны. А если ни один связанный навык вообще не прокачан? value=0? Кажется тогда квесты отображаются не правильно»

Уточнение: «Сейчас показывается как активный. Должен как "не активный"».

## Проблема
В `pers.service.ts` (цикл по `prs.qwests`) квест с `abilitiId` помечался `isNoActive = true` только если `checkTask(abLink.item.tasks[0])` возвращал `false` (т.е. навык неактивен по расписанию/дате).

Но `checkTask` проверяет только дату, а уровень навыка (`Ability.value`) не учитывает. В итоге если навык совсем не прокачан (`value=0`), но расписание сегодня — `checkTask` возвращает `true`, `isNoActive` остаётся `false`, и квест выглядел как активный (без прозрачности, сверху списка).

По смыслу же непрокачанный навык работать не может — связанный квест должен показываться «сереньким» и уезжать в конец, как это сделано в задаче 0002 для неактивных по расписанию квестов.

## Решение
В `src/app/pers.service.ts` в ветке `if (qw.abilitiId)` добавлено дополнительное условие: квест неактивен также если `abLink.item.value` фолси (0/undefined):

```ts
// Квест неактивен, если навык неактивен или не прокачан (value=0)
if (!this.checkTask(abLink.item.tasks[0]) || !abLink.item.value) {
  qw.isNoActive = true;
}
```

Дальше отработает вся уже существующая механика из 0002:
- флаг `isNoActive` пробрасывается в `GlobalItem` (`pers.service.ts:821`)
- сортировка `generateQwestsGlobal` двигает неактивные в конец (`pers.service.ts:827`)
- шаблон `main-window.component.html` применяет `opacity: 0.75`

## Почему такой подход
- Минимальная точечная правка в одной строке — ровно в том месте, где уже выставляется `isNoActive` по связанному навыку.
- Повторно использует всю визуализацию и сортировку из задачи 0002 — никаких новых флагов, полей или шаблонных правок.
- `!abLink.item.value` корректно обрабатывает `0` и `undefined` (по умолчанию в модели `Ability` `value: number = 0`).
