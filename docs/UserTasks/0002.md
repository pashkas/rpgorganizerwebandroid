# 0002 — Показ неактивных квестов с прозрачностью в main-window

## Запрос пользователя
«Квесты, привязанные к навыкам, не показываются если навык неактивен. Можно их показывать, но делать немного прозрачными (75%) и сортировать в конец.»

## Проблема
Квесты с `abilitiId`, чей навык не проходил `checkTask()` (не тот день/расписание), получали `isNoActive = true` и полностью исключались из masonry-сетки на главном окне (`QwestsGlobal`).

## Решение
1. **Условие фильтрации** (`pers.service.ts:2028`): заменили `!qw.isNoActive` на `(!qw.isNoActive || (qw.abilitiId && !qw.parrentId))` — квесты с неактивным навыком теперь попадают в `prs.tasks`, но подквесты (`parrentId`) по-прежнему скрыты.
2. **Модель `GlobalItem`** (`GlobalItem.ts`): добавлено поле `isNoActive: boolean` для проброса флага в шаблон.
3. **`generateQwestsGlobal`** (`pers.service.ts`): пробрасываем `qw.isNoActive` в `GlobalItem`, сортируем массив так, чтобы неактивные шли в конце (стабильная сортировка).
4. **Шаблон** (`main-window.component.html:102`): добавлен `[ngStyle]="{ opacity: qwest.isNoActive ? '0.75' : '1' }"` на `div.main-masonry-item`.

## Почему такой подход
- Аналогичный паттерн с opacity уже использовался в `pers-list.component.html:345` (там 0.4 для неактивных/подквестов).
- Минимальные изменения: 4 файла, точечные правки без рефакторинга.
- Подквесты остаются скрытыми — отсекаются по `parrentId`.

## Затронутые файлы
- `src/Models/GlobalItem.ts`
- `src/app/pers.service.ts` (строки ~2028, ~845)
- `src/app/main-window/main-window.component.html` (строка ~102)
